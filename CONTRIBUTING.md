# Contributing

redtrace accepts contributions that improve authorized engagement tracking,
evidence integrity, reporting, validation, and defensive handoff quality.

Do not contribute:

- exploit payloads;
- phishing templates;
- credential theft logic;
- persistence logic;
- stealth or bypass behavior;
- real target data;
- real client reports.

Preferred contributions:

- report templates;
- safer validation checks;
- evidence integrity improvements;
- lab-safe examples;
- ATT&CK/OWASP/CSF mapping quality;
- terminal UX improvements.
